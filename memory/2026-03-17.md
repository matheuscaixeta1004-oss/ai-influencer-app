## Studio — Sessão da Tarde pt.2 (16:47-17:24 UTC)

### Handles — Saga do clique (5 iterações)
**Problema:** clicar na bolinha arrastava o card em vez de iniciar conexão.
1. Handle cobrir 36px inteiro (não resolveu — card drag interceptava)
2. z-index 50 + CSS `pointer-events: all !important` (não resolveu)
3. Classes `nodrag nopan` no wrapper e Handle (parcial — parou drag mas não iniciava conexão)
4. Bolinhas totalmente fora do card (`-38px` → `-42px`) (parcial)
5. **Solução final:** Handle IS a bolinha — elemento único, sem layers. `<Handle>` renderiza como o círculo visual com ícone dentro (`pointer-events-none`). Offset `-42px` totalmente fora do card.
- **Lição:** em ReactFlow, não usar overlay invisível sobre Handle. O Handle deve SER o elemento visual.

### Drag-to-create corrigido
**Bug:** `pendingConnectionRef` era limpo em `onConnectEnd` antes do menu abrir → `handleMenuSelect` achava null.
**Fix:** ref fica vivo em `onConnectEnd`, só limpo em `handleMenuSelect` após criar o card e edge.
- Simplificado event handling (sem `nativeEvent` extraction)

### Validação de tipo em conexões
- `getHandleDataType(handleId)`: classifica handles em `text`, `image`, `video`, `audio`, `model`, `any`
- `isValidConnection()`: bloqueia conexões cross-type
- ReactFlow dá feedback visual (handle incompatível não aceita durante drag)
- Auto-connect no drag-to-create faz match por tipo
- **Regra:** texto↔texto, imagem↔imagem, vídeo↔vídeo, áudio↔áudio
